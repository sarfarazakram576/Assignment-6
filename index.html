Frequently Asked Questions

The difference between var, let, and const
var: Function-scoped, can be redeclared and updated, hoisted with undefined value, not recommended due to scoping issues
let: Block-scoped, can be updated but not redeclared in the same scope, hoisted but not initialized, safer than var.
const: Block-scoped, cannot be updated or redeclared, must be initialized during declaration, ensures immutability for primitive values.
The difference between map(), forEach(), and filter()
map(): Returns a new array by applying a function to each element without modifying the original array.
forEach(): Iterates over an array and executes a function for each element but does not return a new array.
filter(): Returns a new array containing only the elements that satisfy a given condition.
Explain arrow functions and how they are different from regular functions
Arrow functions are a concise way to write functions using =>. They do not have their own this, instead, they inherit this from the surrounding scope. Unlike regular functions, they cannot be used as constructors and do not have the arguments object. They are useful for short, inline functions and callbacks.
How JavaScript Promises work
JavaScript Promises handle asynchronous operations and have three states: pending, fulfilled, or rejected. A promise executes a task and resolves with .then() on success or .catch() on failure. They help avoid callback hell and improve readability in async code.
How closures work in JavaScript
A closure in JavaScript is a function that remembers and can access variables from its outer scope even after the outer function has finished executing. This happens because inner functions maintain a reference to their parent’s scope. Closures are useful for data privacy, function factories, and maintaining state.
